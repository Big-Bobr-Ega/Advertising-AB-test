# A/B-тест: влияние рекламы на конверсию

Анализ результатов A/B-теста рекламной кампании: оценка
влияния показа рекламы на конверсию в покупку.

## Данные

Датасет [Marketing A/B Testing](https://www.kaggle.com/datasets/faviovaz/marketing-ab-testing)
(Kaggle), 588 101 пользователь.

| Колонка | Описание |
|---|---|
| `user_id` | идентификатор пользователя |
| `test_group` | группа: `ad` (реклама) или `psa` (контроль) |
| `converted` | совершил покупку (0/1) — целевая метрика |
| `total_ads` | число показанных объявлений |
| `most_ads_day` | день недели с максимумом показов |
| `most_ads_hour` | час суток с максимумом показов |

Сплит групп: 96% `ad` (564 577) / 4% `psa` (23 524).

## Бизнес-контекст

Тестовая группа `ad` видит рекламу, контрольная `psa` — нейтральную заглушку (public service announcement) на том же месте. Цель — определить,
добавляет ли реклама покупки или пользователи купили бы и без неё.

## Что исследуется

1. Повышает ли реклама конверсию: $H_0: p_{ad} \le p_{psa}$ против $H_1: p_{ad} > p_{psa}$
   (односторонний тест, $\alpha = 0.05$).
2. Есть ли причинная связь между числом показов и конверсией.

## Методы

- EDA и чистка данных (типы, дубликаты, пропуски, выбросы).
- A/A-тест для валидации процедуры (hash-сплит).
- z-тест для пропорций и хи-квадрат (перекрёстная проверка, $\chi^2 = z^2$).
- Величина эффекта: абсолютная разница, относительный lift, Cohen's h; доверительные интервалы.
- Расчёт мощности и MDE (statsmodels).
- Байесовский A/B (сопряжённая модель Beta-Binomial).
- Анализ confounding (сравнение конверсии по числу показов внутри групп).
- SQL-агрегации (DuckDB).
- Визуализация (matplotlib).

## Результаты

- Конверсия: `ad` 2.55% против `psa` 1.79%.
- Эффект: +0.77 п.п. (ДИ [0.59, 0.94] п.п.), +43% относительно контроля (ДИ [30%, 57%]).
- Значимость: z = 7.4, p ≈ 8.5e-14; $\chi^2$ = 54; $P(ad>psa) \approx 1$.
- Причинность: эффект группы `ad`/`psa` — причинный. Связь «больше показов → выше конверсия»
  проявляется и в контроле `psa` без настоящей рекламы, то есть это confounding (вовлечённость),
  а не эффект рекламы.
- Рекомендация: раскатывать рекламу; при повторном тесте сбалансировать сплит.

## Стек

Python (pandas, numpy, scipy, statsmodels), DuckDB(SQL), matplotlib
